智能化环境管理系统

环境管理痛点和解决方案

项目背景和痛点

传统的环境管理系统，存在以下问题：

  1. 管理人员需要手动录入数据，效率低下；
  2. 环境分层多，查找困难，管理复杂；
  3. 管理人员需要逐个手动处理数据，容易出错，效率低；
  4. 管理人员测试过程中手动发现环境问题，效率低下
  5. 无AI技术支持，无法自动化理解和处理数据；
  6. 对固化环境管理弱，个人线下进行，信息不畅，且易丢失；
    如何实现AI技术驱动的环境管理系统？对AI更加友好的环境管理系统？如何实现对AI的优化和集成？是我们当前的开发重点。

环境管理解决方案和创新点

  1. 解耦设计；
    将环境管理系统拆分为多个子系统，每个子系统负责不同的功能，比如环境配置系统、环境监控系统、数据分析系统等，每个子系统之间通过消息队列进行通信，实现解耦设计，大幅提升扩展性和可维护性。
  2. Ai友好设计；
    在环境配置系统中，每个环境配置，直接简化成一个JSON配置，所有的设备配置是一个小的JSON配置，大幅提升对AI的友好型。同时使用AI技术进行环境配置、环境检查，也可以分析监控数据，生成报告，实现对AI的友好支持。
  3. 智能监控；
    环境使用过程中，自动对环境进行监控，上报设备信息和环境状态，关联执行的测试任务和责任人，实现智能监控。并且后台并行处理，实现高性能。
  4. 极简设计；
    没有复杂分层和复杂展示的页面，都只是一行的记录数据，一次性可以管理多个环境，整体环境信息全景查看，快速分析；单环境直接查看，一键点击权力查看；环境更新配置，一键完成；
  5. 自适应的前端展示；
    设备参数信息进行分层设计，在线统一管理，可以实现自由添加设备和进行参数编辑、排序等操作；使用vue3.0的响应式系统，实现自适应的前端展示，无需后端开发人员参与，实现前后端分离。

项目设计

项目技术栈

springboot + mybatis-plus + mysql8 + redis + mqtt + deepseek
MQTT（Message Queuing Telemetry Transport，消息队列遥测传输）是一种轻量级的发布/订阅（Pub/Sub）消息协议，专为低带宽、网络不稳定、计算能力有限的设备设计。

项目难点

如何设计数据采集和处理流程？
如何实现数据采集和处理？
如何实现数据展示？多样化的环境，如何进行灵活管理，降低运维？
如何实现数据分析？
如何实现数据存储？
如何设计对AI友好的数据存储？
如何实现AI技术支持，提升环境管理效率？

项目实现

  1. 数据采集：使用执行器采集环境数据，并存储到数据库中
  2. 数据处理：使用AI技术进行数据处理，分析数据，生成报告
  3. 数据展示：使用可视化工具展示数据，分析结果

项目架构

  1. 使用微服务架构，实现高可用和高性能
  2. 使用SpringBoot开发API，实现数据采集
  3. 使用SpringBoot开发Web界面，实现数据采集
  4. 使用SpringBoot开发移动端应用，实现数据采集
  5. 使用SpringBoot开发桌面端应用，实现数据采集
  6. 使用mysql存储数据，其中使用一个大JSON字段存储环境数据，使用一个JSON字段存储AI分析结果
  7. 使用vue开发Web界面，实现环境信息的展示和分析
  8. 使用redis存储数据，用于缓存和存储AI分析结果

数据采集

  1. 使用执行器采集环境数据，并存储到数据库中
  2. 使用MQTT协议，实现数据采集。
  3. 使用SpringBoot开发API，实现数据采集。
  4. 使用vue开发Web界面，实现数据展示。

数据展示

  1. 使用vue开发Web界面，实现环境信息的展示和分析。
  2. 使用echarts展示数据。

设备的管理

环境中的设备包含仪表、DUT、BBU、电源等。
仪表的分类有：示波器、频谱仪、信号源、电源
不同分类下又有不同给品牌型号：频谱仪包含FSQ、FSV、FSW等；信号源包含：SMU200A、SMW200A等；

数据处理

  1. 使用AI技术进行数据处理，分析数据，生成报告。
  2. 使用deepseek进行数据处理，分析数据，生成报告。
