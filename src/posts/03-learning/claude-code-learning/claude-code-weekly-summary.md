## 📚 学习历程

### 第一阶段：Hooks 机制（day7_hooks）

**核心问题：** 如何在不修改核心代码的情况下扩展功能？

**答案：** Hook（钩子）机制。

**三种 Hook 类型：**

| 类型 | 时机 | 用途 |
|------|------|------|
| Pre-Hook | 执行前 | 参数验证、权限检查 |
| Post-Hook | 执行后 | 日志记录、结果处理 |
| Error-Hook | 出错时 | 错误处理、重试机制 |

**一句话精髓：**
> Hook = 在关键点插入自定义逻辑，不改核心流程

**Hook vs 插件：**

| | Hook | 插件 |
|--|------|------|
| 重量 | 轻量（单点） | 重型（完整模块） |
| 典型用途 | 日志、校验 | 微信、飞书 |

### 第二阶段：Memory 管理（day8_memory）

**核心问题：** Agent 如何跨会话记住信息？

**两种 Memory 类型：**

| 类型 | 作用 | 持久化 |
|------|------|--------|
| 短期记忆 | 会话内消息历史 | 内存，会话结束消失 |
| 长期记忆 | 跨会话持久化 | 文件/数据库 |

**Memory 的读写时机：**
- **读取：** 用户输入时，加载到上下文
- **写入：** 执行过程中，重要信息持久化

**代码实现要点：**
```python
class Memory:
    def save(self, key, value):
        # 持久化到文件
        self.data[key] = value
        json.dump(self.data, open(self.path, "w"))
    
    def load(self, key, default=None):
        # 从文件读取
        return self.data.get(key, default)
    
    def build_context(self):
        # 构建筑上下文字符串，注入到 system prompt
        return "【记忆】" + " | ".join([f"{k}: {v}" for k, v in self.data.items()])
```

**追问：** 关键词匹配识别重要信息的局限性？
- 硬编码，不灵活
- 无语义理解
- 泛化能力差

**更好的方案：** 用 LLM 判断是否重要，或用向量检索（RAG）

### 第三阶段：MCP 协议（04-15）

**核心问题：** 如何让 Agent 调用外部服务？

**答案：** MCP（Model Context Protocol）

**MCP 架构：**
```
Agent → MCP Client → MCP Server → 外部服务
                          ↓
              工具定义注册 + 请求转发
```

**MCP vs 普通工具：**

| | 普通工具 | MCP |
|--|----------|-----|
| 定义位置 | 代码 | 配置文件 |
| 调用方式 | 直接函数调用 | HTTP/WebSocket |
| 扩展性 | 低 | 高 |
