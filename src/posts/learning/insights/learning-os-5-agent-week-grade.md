# 学习操作系统升级 — 5 Agent 协作 + 周分级思维 + 极简哲学

## 💎 一句话总结

> 把"用 Agent 帮我学习"从一句话变成**可执行的工业级流程**：
> **5 Agent × 周分级 × 极简哲学 = 70 篇文章不是目的，掌握才是**。

***
## 🎯 三大升级（一句对比）

| 维度 | 旧版（4 Agent）| 新版（5 Agent）|
|------|---------------|----------------|
| 协作 | 4 步循环 + 双适用 | 5 步：Week Review 加入 |
| 难度 | 全程一样 | 🌱 W1 → 🌿 W2-3 → 🌳 W4-6 |
| 文档 | V2/V3/V4 历史 + 9×3 思维表 | 极简：5 步 + 难度表 |

***
## 🆕 升级 1：5 Agent 协作（4 Agent → 5 Agent）

### 新增 Week Review Agent

| Agent | 角色 | 频率 | 输出 |
|-------|------|------|------|
| Content | 每日学习教练（宏→微递进）| 每天 | content/day-{n}.md |
| Exam | 出考题（难度按周递进）| 每天 | exams/exam-day-{n}.md |
| Error | 错题根因分析 | 按需 | errors/error-day-{n}.md |
| Meta | 颜色地图追踪 | 每天 | state.json |
| **Week Review** | **周复盘：分数 + 规律 + 建议** | **每周** | **reviews/week-{n}.md** |

### 5 步循环 = 完整闭环

```
每日 5 步：
  Content → Exam → 你做题 → Error → Meta
每周加 1 步：
  + Week Review（Day 7 触发）
```

### Week Review 输出（5 部分）

```
1. 📊 分数趋势（表格 + 迷你折线图）
2. 🎨 颜色变化（迁移路径）
3. ❌ 错题规律 Top 3（4 大根因归类）
4. 🎯 下周建议（是否调整 + 重点 + 策略）
5. 💡 一句话总结
```

## 📊 升级 2：周分级思维（核心创新）

### 为什么不"一视同仁"？

> **反事实思考**：如果全部都用第一性原理（最难的思维），新学者会怎样？
> 答：会"过度推理"——把简单问题搞复杂，陷入"哲学陷阱"。
>
> **正确做法**：让**思维深度**和**学习阶段**对齐。

### 三阶段全表

| 周 | 阶段 | 主导思维 | 考题深度 | 时长 |
|----|------|---------|---------|------|
| 1 | 🌱 熟悉 | 抽象建模 + 类比 | 宏观定位 + 填空 | 15-20 min |
| 2-3 | 🌿 深入 | +推论假设 + 归纳 + 批判 | 复现 + 改错 | 20-30 min |
| 4-6 | 🌳 精通 | +第一性 + 反事实 + 系统 | 改写 + 场景 + 反事实 | 30-45 min |

### 🌱 W1 特殊规则（最重要）

- ❌ **禁止反事实题**（"如果...会怎样"）— 用户还在认路
- ❌ **禁止跨模块综合题** — 单模块单文件
- ✅ **每道题必须 2 分钟内答完** — 降低认知负担
- ✅ **优先填空题**而非设计题 — 减少"答不出"挫败感
- ✅ **先宏观后微观**（30%+40%+30% 出题顺序）

### 🌳 W4+ 考试结构（对照参考）

| 题型 | 思维深度 | W1 可用？| 示例 |
|------|---------|---------|------|
| 复述题 | L1 记忆 | ✅ | "一句话说 X" |
| 填空题 | L1-L2 理解 | ✅ | "Msg 的 8 个字段" |
| 复现题 | L2 理解 | ✅ | "创建 UserMsg 的代码" |
| 改错题 | L2-L3 应用 | ✅（简单错误）| "哪里不对？" |
| 改写题 | L3 应用 | ❌ W2+ | "重构这段..." |
| 反事实题 | L4 分析 | ❌ W4+ | "不这样设计会怎样？" |
| 场景题 | L5 创造 | ❌ W4+ | "解决新问题..." |

## 📐 升级 3：极简哲学（删繁就简）

### 删除项（"少即是多"）

- ❌ V2/V3/V4 版本对比表（混乱源头）
- ❌ `initial-state.md`（state.json 才是事实源）
- ❌ 9×3 思维清单（太多，删不掉）
- ❌ USA[G]E.md 详尽说明（迁到 tools/）

### 极简原则

> **每个文件只回答 1 个问题**：
> - README = "这是什么"
> - USAGE.md = "怎么用"
> - tools/* = "实现细节"
> - agents/* = "如何思考"
> - codebases/ = "学了什么"

***
## 🔍 反事实推理：为什么必须升级？

### 不升级会怎样？

> **反事实 A**：如果还是 4 Agent，没有周复盘：
> → 学完 1 周 = 零散 7 天经验，没法看趋势
> → 一道错题可能下周又错

> **反事实 B**：如果还是"全期统一思维"：
> → Day 1 用户就看到"反事实"题，吓跑
> → 学得"压抑"而非"渐进"

> **反事实 C**：如果还有 V1/V2/V3 多个版本：
> → 谁都不知道"该用哪个"
> → 主版本 = 现在的 README，是唯一正解

## 🔁 横向对比：相似系统

| 系统 | 与本系统的差异 |
|------|---------------|
| AutoGen / MetaGPT | 偏向多 Agent **协作做任务**，不是协作**学习** |
| 极客时间专栏 | **单向**输出，不是"考试验证"闭环 |
| 大学课程 | **周复盘**对了，但缺元认知自评 |
| 学习博主流程 | 单一视角，本系统是"导师 + 学生 + 评分员"三方协作 |

**本系统独特价值**：**W1 → W6 思维深度自适应**（他人都没做）。

## ⚙️ 实现关键（架构师视角）

### 5 大硬约束（防止系统被错用）

1. **路径硬约束**：每个 Agent 只能写**精确路径**
   ```
   Content → content/day-{n}.md
   Exam    → exams/exam-day-{n}.md（含作答区）
   Error   → errors/error-day-{n}.md
   Meta    → colors/color-day-{n}.md + state.json
   Week    → reviews/week-{n}.md
   ```

2. **Day 0 vs Day 1+ 区分**：初始化和每日用不同 Agent 模式

3. **周分级思维**：按 Week 选用思维，强行跳过 W1 反事实

4. **考前错题回顾**：Content Agent 必填"昨日回顾"段

5. **作答区域模板**：Exam Agent 必含"✏️ 你的作答"

### 一次完整使用流程（手动模式）

```bash
# Day N 每日（Day 1-6）
Agent 1: 复制 agents/content-agent.md 给 LLM
        输出 → content/day-{n}.md
Agent 2: 复制 agents/exam-agent.md + 上一步输出
        输出 → exams/exam-day-{n}.md
你:     在 exam 的"✏️ 你的作答"区域写答案
Agent 3: 复制 agents/error-agent.md（如有错题）
        输出 → errors/error-day-{n}.md
Agent 4: 复制 agents/meta-agent.md + 今日全部数据
        输出 → 颜色更新到 state.json

# Day 7 加一步
Agent 5: 复制 agents/week-review-agent.md + 本周数据
        输出 → reviews/week-{n}.md

# Day 0 一次性
Agent 1: 复制 agents/content-agent.md + 代码库信息
        输出 → src-map.md + learning-plan-6weeks.md
```

## 💎 核心哲学：3 条铁律

### 1. **"问 ≠ 会"**

> 必须用考试验证掌握度，不能"我看了文档 = 我会了"。

### 2. **"看过 ≠ 理解"**

> 9 大思维让你从"看过"到"理解设计哲学"。
> 抽象建模 → "它是什么形状"
> 类比推理 → "它和谁类似"
> 第一性原理 → "它为什么是它"

### 3. **"元认知 = 对认知的认知"**

> 知道自己**知道什么** + **怎么知道的** + **不知道什么**。
> 这不是口号，是**系统可观测**的事实。
