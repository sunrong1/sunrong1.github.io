## 🎯 核心考点精要

### 1. 关键路径法（CPM）

**考点：** 计算项目最短工期、识别关键活动

**公式：**
- 关键路径 = 最长路径 = 最短工期
- 总浮动时间 = 最晚开始 - 最早开始
- 关键活动的总浮动时间 = 0

**实践应用：**
```
在自动化测试平台项目中，我使用关键路径法识别出：
- 测试框架设计（5 天）→ 关键路径
- 用例开发（10 天）→ 关键路径  
- 环境搭建（3 天）→ 非关键路径（有 7 天浮动）

通过并行执行非关键活动，整体工期从 18 天压缩到 15 天。
```

### 2. 挣值管理（EVM）

**三大核心指标：**

| 指标 | 公式 | 含义 |
|-----|------|-----|
| PV（计划价值） | 计划完成工作的预算 | 应该干多少 |
| EV（挣值） | 实际完成工作的预算 | 实际干了多少 |
| AC（实际成本） | 实际花费的成本 | 实际花了多少 |

**绩效指数：**
- **SPI = EV / PV** （进度绩效指数，>1 表示进度超前）
- **CPI = EV / AC** （成本绩效指数，>1 表示成本节约）

**典型考题：**
```
项目预算 100 万，计划 10 个月完成。第 6 个月时：
- PV = 60 万（计划完成 60%）
- EV = 50 万（实际完成 50%）
- AC = 70 万（实际花费 70 万）

计算：
SPI = 50/60 = 0.83（进度落后）
CPI = 50/70 = 0.71（成本超支）

结论：项目进度落后且成本超支，需要采取纠正措施。
```

### 3. 风险管理

**风险应对策略：**

| 策略 | 适用场景 | 示例 |
|-----|---------|-----|
| **规避** | 高影响高概率 | 更换技术方案 |
| **转移** | 高影响低概率 | 购买保险、外包 |
| **减轻** | 中影响中概率 | 增加测试、冗余设计 |
| **接受** | 低影响低概率 | 建立应急储备 |

**我的实践：**
在可靠性测试 AI 机器人项目中：
- **技术风险**（高概率高影响）→ 规避：选择成熟框架
- **进度风险**（中概率中影响）→ 减轻：增加缓冲时间
- **人员风险**（低概率高影响）→ 转移：建立 AB 角

## 💡 敏捷项目管理

### 敏捷 vs 预测型

| 维度 | 预测型（瀑布） | 敏捷型 |
|-----|--------------|-------|
| 需求 | 前期明确 | 迭代演进 |
| 变更 | 严格控制 | 欢迎变更 |
| 交付 | 一次性交付 | 增量交付 |
| 客户参与 | 阶段性 | 持续参与 |
| 适用场景 | 需求稳定 | 需求不确定 |

### 敏捷核心实践

1. **每日站会** - 15 分钟同步进度和障碍
2. **迭代规划** - 2-4 周一个迭代
3. **回顾会议** - 持续改进
4. **用户故事** - As a... I want... So that...
5. **燃尽图** - 可视化进度

### 混合式项目管理

在实际工作中，我采用**混合式方法**：
- **高层规划**：预测型（年度规划、里程碑）
- **执行层面**：敏捷型（迭代开发、持续交付）
- **质量管理**：预测型（测试策略、验收标准）

### 时间规划（3 个月）

```
第 1 个月：精读 PMBOK，建立知识框架
第 2 个月：刷题巩固，查漏补缺
第 3 个月：模拟考试，冲刺突破
```

### 考试技巧

1. **审题关键**：注意"除了"、"首先"、"最佳"等关键词
2. **情景题**：站在 PM 角度思考，不要代入个人经验
3. **计算题**：熟练掌握 EVM、关键路径公式
4. **时间分配**：180 分钟 180 题，平均 1 分钟 1 题
