主题一：Agent Loop

主循环逻辑

Agent 的"思考引擎"

核心流程

用户输入 → 理解意图 → 规划行动 → 调用工具 → 接收反馈 → 输出结果
     ↑__________________________________________________↓
                    （循环直到任务完成）

为什么需要循环？

🤖 普通 API = 一次调用 = 问什么答什么

🔄 Agent Loop = 循环调用 = 自主完成任务

类比：做饭

  • 普通 API：给你食材，告诉你做什么
  • Agent Loop：帮你做完整顿饭（洗菜→切菜→炒菜→装盘）

代码实现

class AgentLoop:
    def run(self, user_input, max_turns=10):
        for turn in range(max_turns):
            response = self.model.generate(prompt)  # 调用模型
            
            if response.is_complete():  # 判断是否完成
                return response.final_answer()
            
            if response.needs_tool():  # 需要调用工具
                tool_result = self.execute_tool(response.tool_call)
                self.memory.add_turn(...)  # 记忆更新
        
        return "任务超时"

主题二：Hook

钩子机制

关键节点的"拦截器"

Hook 类型

类型作用类比
before_tool工具执行前拦截海关检查
after_tool工具执行后拦截快递签收
before_model模型调用前拦截考试前检查
after_model模型调用后拦截批卷检查

应用场景

✅ 参数校验 - before_tool：检查参数是否安全

✅ 日志记录 - after_tool：记录工具执行结果

✅ 安全检查 - before_model：检查 prompt 注入

✅ 输出过滤 - after_model：检查输出合规

代码实现

class HooksManager:
    def execute_hooks(self, hook_type, context):
        for hook in self.hooks[hook_type]:
            result = hook(context)
            if result.get('skip'):  # 拦截
                return result
        return context

# 使用
hooks.register('before_tool', validate_params)
hooks.register('after_tool', log_result)

主题三：Memory

记忆系统

分层记忆管理

为什么需要分层？

⚠️ 上下文窗口有限 - LLM 有 token 限制

⚠️ 不是所有信息都重要 - 需要筛选

⚠️ 成本考量 - token 越多费用越高

三层记忆架构

┌─────────────────────────────────────────┐
│  瞬时记忆  │  当前上下文      │ 随token淘汰 │
│  短期记忆  │  本次会话历史    │ 关闭后消失 │
│  长期记忆  │  持久化存储      │ 跨会话保留 │
└─────────────────────────────────────────┘

主题四：Planning

任务规划

复杂任务的分解与执行

规划策略

策略场景例子
单步规划简单任务"关闭 BBU"
多步规划复杂任务"配置环境并测试"
递归规划不确定步骤Agent 自决定

ReAct 模式

Thought → Action → Observation → Thought → ...
 思考      行动       观察        思考

示例：
Thought: 需要先查询当前配置
Action: query_env_config
Observation: 查到了 2 个 BBU，3 个 RRU
Thought: 下一步设计配置方案
Action: design_env_config
...

代码实现

def planning_react(task, tools):
    for step in range(max_steps):
        thought = model.generate(thought_prompt)
        
        if 'finish' in thought:
            return result
        
        action, params = parse_plan(thought)
        observation = tools[action].execute(params)
    
    return "任务未完成"

主题五：MCP

Model Context Protocol

标准化 Agent 与工具的交互

MCP 是什么？

🔌 类比：电源适配器

  • 不管什么设备（手机、电脑）
  • 都用标准电源接口
  • 适配器负责转换

💡 MCP = Agent 的"适配器协议"

架构图

┌──────────────┐       MCP        ┌──────────────┐
│   LLM/Agent  │ ←─────────────→  │  MCP Server  │
│              │                  │              │
│  - prompts   │                  │  - tools     │
│  - resources │                  │  - resources │
│  - tools     │                  │  - limits    │
└──────────────┘                  └──────────────┘

MCP Server 示例

from mcp.server import MCPServer

hero = MCPServer(name="HERO")

@hero.tool()

            'rrus': ['RRU-1', 'RRU-2']}

hero.run()

主题六：Harness Engineering

工程化编排

给 LLM 戴上马具

核心问题

❌ 没有 Harness 的 LLM：

  • 会胡说八道
  • 乱调用工具
  • 忘记关键上下文
  • 行为不可预测

解决方案

🐎 Harness = 马具

野马很强大，但无法直接使用。戴上马具后才能拉车、耕地。

🤖 LLM = 野马，Harness Engineering = 给 LLM 戴上马具

架构图

┌───────────────────────────────────────┐
│        Harness Engineering             │
├───────────────────────────────────────┤
│  Planning ──→ Hooks ──→ Memory ──→ Tools │
│       ↑                                 │
│       └────── Feedback Loop ───────────┘│
└───────────────────────────────────────┘

LLM 是大脑，Harness 是神经系统

实战演示二

Memory 组件

代码实现

class HeroMemory:
    def __init__(self, user_id):
        self.short_term = []
        self.long_term = self._load()
    
    def add_interaction(self, user, agent, tools):
        self.short_term.append({
            'user': user,
            'agent': agent,
            'tools': tools
        })
        if len(self.short_term) > 10:
            self.compress()
    
    def compress(self):
        # 保留最近 4 轮 + 摘要
        summary = self._summarize(self.short_term[:-8])
        self.short_term = [{'type':'summary', 'content':summary}] 
                           + self.short_term[-8:]
        self._save_long_term(summary)

# 演示：对话历史记住
memory.add_interaction("帮我配置 RRU-1", "...", ['design_env_config'])
memory.add_interaction("改成 RRU-2", "...", ['design_env_config'])
# Agent 能记住上一轮是 RRU-1
